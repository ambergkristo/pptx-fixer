# CleanDeck — Truth Reset Roadmap and Milestone Statute

## 1. Official product truth

Until further proof, CleanDeck is officially positioned as:

**Audit-first deck QA tool with safe partial normalization**

Not yet:
- strict cleanup engine
- template enforcement platform
- AI deck post-processing layer

## 2. Why this reset exists

The current product has a strong report layer, strong UI shell, corrected PPTX export, and credible font/size normalization. However, hostile stress testing still indicates unresolved closure on paragraph-level cleanup classes. Therefore:
- UI maturity must not be confused with engine closure
- all future milestones must prioritize proof before expansion
- future strategy must remain tied to demonstrated capability

## 3. Core operating rules

### Scope rules
CleanDeck does:
- audit existing PPTX files
- apply safe autofix where confidence is high
- preserve structure and editability
- export corrected PPTX
- produce deterministic reports

CleanDeck does not do in current roadmap phases:
- slide generation
- redesign / re-layout
- narrative rewrite
- AI deck generation
- broad template enforcement before engine closure proof

### Control rules
For every sprint:
1. ChatGPT defines the sprint prompt
2. Codex implements only that sprint
3. Codex returns a structured summary
4. ChatGPT reviews PASS / FAIL
5. No next sprint starts before review
6. ChatGPT and Codex must both re-check this roadmap before every sprint

### Drift prevention rules
No sprint may:
- redefine product truth silently
- expand beyond active milestone scope
- introduce late-phase product ideas early
- treat polished UI as proof of engine readiness

## 4. Acceptance truth

The MVP is only considered proven when category-level evidence exists for all six core cleanup classes:
- font normalization
- size normalization
- alignment cleanup
- bullet / indent cleanup
- line spacing cleanup
- paragraph spacing cleanup

Stress-test and hostile decks are mandatory acceptance inputs.
Manual review must become the exception for eligible decks, not the default outcome.
